# Plan: PressScribe — notes UI and autosave workflow

## Цель

Сделать единый и простой UX для текстовых заметок в PressScribe:
- заметки сохраняются автоматически по умолчанию
- автосохранение можно выключить кнопкой/переключателем в настройках (включено по-умолчанию)
- список заметок идёт от новых к старым
- заметку можно открыть, редактировать и копировать
- поддержать удаление одной заметки, нескольких заметок и всех заметок
- сохранить одинаковую логику для mobile Android и desktop, насколько это позволяет кодовая база

## Что уже есть сейчас

По текущему коду и документации видно, что проект уже умеет:
- desktop transcription + polishing workflow
- сохранение/открытие сессий
- Android-версию со схожим split workflow
- само-хостed ASR и облачные/локальные варианты в зависимости от настроек

То есть: вносить правки и изменения в текущий форклоу именно создания заметок и редактирование заметок в основном экране не требуется, нужно добавить новый экран где будут хратится автосохраненные заметки. 

## Основная идея UX

### 1. Экран заметок
Сделать отдельный экран/раздел "Saved notes".

На нём:
- список заметок
- сверху — самые новые
- каждая заметка в списке показывает:
  - заголовок из первых слов
  - короткий превью-фрагмент
  - кнопку/иконку copy
  - чекбокс или режим выбора для массового удаления

### 2. Открытие заметки
При нажатии на элемент списка открывается экран заметки:
- полный текст доступен для редактирования
- отдельная кнопка copy для текста целиком
- понятная кнопка back / return to list

### 3. Автосохранение
По умолчанию:
- автосохранение включено
- каждый новый текстовый результат сохраняется автоматически

Нужен переключатель:
- "Auto-save"
- on by default
- если выключен, пользователь сам решает, когда сохранять - добавить кнопку-иконку на той панели где кнопки Copy/Clear/Clear All. Логика ручной кнопки сохранения - по нажанию заметка сохраняется. пока не будет очищена панель от текущей замеки, каждое нажатие сохранить обновляет текущую сохраненную замеки (а не создает новую).

### 4. Навигация жестом
Пользователь хочет поведение, похожее на swipe:
- в mobile: свайп влево/вправо для перехода между экранами
- в desktop: место свайпа просто преключение интерфейса от экрана создания замеки к экрану списка сохраненных заметок и назад. 

## Функции управления заметками

### В списке
- сортировка по `created_at` от новых к старым
- copy-иконка у каждой записи
- delete-иконка у каждой записи
- multi-select mode
- delete selected
- delete all
- подтверждение перед массовым удалением

### Внутри заметки
- редактирование текста
- copy-иконка - copy full text
- сохранение изменений обратно в ту же запись

## Структура данных

Для каждой заметки нужны минимум:
- `id`
- `title` — первые слова текста (примерно в ширину экрана устройства)
- `content`
- `created_at`
- `updated_at`
- `source` или `origin` при необходимости
- `pinned` только если потом понадобится, сейчас не обязательно

## Что важно не перепутать

- Это не список транскрипций, а заметка на основе вывода панели "Polished text"
- Автосохранение должно быть **включено по умолчанию**.
- Copy должен быть доступен и в списке, и внутри заметки.
- Новые записи должны быть сверху.
- Удаление нескольких записей должно быть удобным и безопасным.
